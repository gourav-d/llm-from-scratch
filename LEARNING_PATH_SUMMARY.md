### Dependencies Explained

**Why Prompt Engineering First?**
- No infrastructure needed
- Immediate value
- Foundation for RAG
- Cheapest optimization
- Essential for all modules

**Why Vector DBs Before RAG?**
- RAG requires vector search
- Simpler in isolation
- Reusable skill
- Build intuition first

**Why RAG Before LangChain?**
- Understand the pattern first
- LangChain is a tool, RAG is a concept
- Better appreciation of framework value
- Can build RAG without LangChain

**Why Fine-Tuning Later?**
- More complex than RAG
- Requires GPU resources
- Often RAG is sufficient
- Advanced optimization technique

**Why LLMOps Before Security?**
- Need deployment knowledge first
- Security builds on LLMOps
- Monitoring enables security
- Holistic view

**Why Security Before Multi-Modal?**
- Security is more urgent
- Must design from start
- Multi-modal amplifies security risks
- Foundation before expansion
